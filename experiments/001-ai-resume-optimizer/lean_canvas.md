# Lean Canvas · AI 简历优化器

> 来源 ideas/2026-06-25 #1（首选，RICE 7.2）。填于 2026-06-30，experiment 001。

## 填表前：第一性原理门 ✅ 通过
- 去假设：去掉"AI工具都赚钱"，剩下事实——求职竞争长期存在、简历是求职第一关。
- 底层需求：**求职竞争焦虑**（持续、真实、强，每年数千万求职者）。
- 最小可证事实：人工简历优化是成熟付费品类（$50-300/次），付费意愿已被市场验证；ATS 过滤 ~75% 简历是公认痛点。
- 反共识/edge：红海，技术无壁垒。唯一可能 edge 见第 5 块。
- 门结论：✅ 通过。**实测核心问题不是"能不能做"，而是"在不投流/不买量前提下，能否靠自然内容流量拿到第一笔¥9.9"。**

## 1. 问题（top 3）
1. 简历"罗列职责"而非"量化成果"，HR 6 秒扫视 + ATS 双重过滤下被刷
2. 不会针对具体 JD 关键词定制，过不了 ATS 筛选
3. 不知道自己简历哪儿弱、改往哪个方向

## 2. 现有替代方案
- 人工简历顾问：$50-300，慢，质量参差
- 通用 AI（直接问 ChatGPT）：免费但不懂 JD 定制、不懂量化写法、需反复 prompt
- 招聘平台自带模板（Boss/智联）：格式有，内容优化弱

## 3. 解决方案（top 3）
1. 上传旧简历 + 贴 JD → 一键输出针对该 JD 的重写版（量化成果 + 关键词匹配）
2. ATS 通过率评分（指出缺哪些关键词）
3. 同段经历多版本 A/B（3 种写法）

## 4. 关键指标
- 每日上传简历数（需求信号）
- **fake-door"¥9.9 解锁导出"点击率**（核心转化信号，先测这个）
- 7 日留存（是否针对多个 JD 回来）
- 单用户日均生成次数

## 5. 不公平优势（诚实）
- 技术壁垒：**弱/无**（AI wrapper）。
- 可能的 edge：① 用户自身多领域身份（量化/销售/健康/独立开发）→ 做"细分岗位定制"（如"量化研究员简历""B2B 销售简历"），细分到 大厂不愿做；② ai-money-lab 公开仓库 + 知乎/飞书渠道可冷启动。
- ⚠️ 若这两条都跑不出转化 → 无真 edge，归"没赚/不可复刻"，进失败榜。

## 6. 渠道（冷启动）
- 知乎/小红书"简历优化"长尾问题截流（每篇带工具链接）
- ai-money-lab 仓库 README 导流
- 后续：脉脉/Boss 直聘求职社群

## 7. 客户细分
- 目标：应届 + 3 年内职场新人（简历焦虑最强、决策快）
- 理想客户：海投党、转行党、面试季在校生
- 次优先：细分岗位（量化/销售）求职者（吃 edge）

## 8. 成本结构
- 固定：域名 ~$10/年、Vercel 部署 $0
- 变动：Claude API ~$0.05/次（随用量线性）
- 时间：MVP 2-3 天（Claude Code 辅助），日均维护 0.5h
- 隐性大头：导流内容创作时间

## 9. 收入来源
- 主：单次 ¥9.9 解锁高质量导出（PDF/Word）——低门槛冲动付费
- 增：¥29 包月（无限次针对不同 JD）
- 测法：**fake-door 先测"¥9.9 解锁"点击率，>3% 再接真支付（爱发电/Stripe）**——避免先建支付再发现没人要

---
_关键判定：渠道(6) + 不公平优势(5) 两块若被数据证伪 → 归失败。这不是"做不做得出来"，是"导不导得到第一笔"。_
